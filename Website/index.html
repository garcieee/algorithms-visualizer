📖 Algorithm Explanations & Complexity
🫧 Bubble Sort — O(n²) avg · O(n) best · O(1) space ▼
Repeatedly compares adjacent pairs and swaps them if out of order. Each pass "bubbles" the largest unsorted element to the end. Early-exit optimization: stops immediately when a full pass makes zero swaps — meaning the array is already sorted.
CaseTimeWhen
BestO(n)Already sorted (early exit fires)
AverageO(n²)Random input
WorstO(n²)Reverse sorted

Data Structure: In-place array. Two index variables j and j+1 walk the array.

🎯 Selection Sort — O(n²) all cases · O(1) space ▼
Each pass scans the entire unsorted portion to find the minimum element, then places it at the start. Always makes exactly n-1 swaps regardless of input — good when swaps are expensive.

Data Structure: In-place array. One index pointer tracks the current minimum.

📌 Insertion Sort — O(n²) avg · O(n) best · O(1) space ▼
Like sorting a hand of playing cards: pick each element and insert it into the correct position in the already-sorted prefix. Very efficient on nearly-sorted data. Stable sort (equal elements keep their original order).

Data Structure: In-place array. A "key" variable holds the current element being inserted.

🔀 Merge Sort — O(n log n) all cases · O(n) space ▼
Divide-and-conquer: recursively split the array in half, sort each half, then merge the two sorted halves. Guaranteed O(n log n) on all inputs. Stable sort. Uses auxiliary arrays for merging.

Data Structure: Auxiliary arrays for each merge step. Recursion call stack O(log n) deep.

⚡ Quick Sort — O(n log n) avg · O(n²) worst · O(log n) space ▼
Picks a pivot, partitions elements so everything ≤ pivot is on the left and > pivot on the right, then recursively sorts both sides. Worst case O(n²) occurs when pivot always picks the min or max (e.g., on sorted input with last-element pivot).

Data Structure: In-place array partitioning. Two index pointers i and j.

🎰 Randomized Quick Sort — O(n log n) avg · O(log n) space ▼
Same as Quick Sort but chooses the pivot randomly before partitioning. This eliminates the worst-case on sorted/reverse-sorted input — the probability of hitting O(n²) becomes astronomically small.

Data Structure: Same as Quick Sort. Random pivot is swapped to the end before partitioning.

🔢 Counting Sort — O(n+k) · O(k) space · Non-comparison ▼
Counts how many times each value appears, then reconstructs the sorted array. Not comparison-based — breaks the O(n log n) lower bound. Only works on non-negative integers. k = max value in the input.

Data Structure: Count array (frequency table) of size k. Rebuilt output array.

📻 Radix Sort — O(nk) · O(n+k) space · Non-comparison ▼
Sorts by digit position, from least significant to most significant (LSD). Each digit pass uses counting/bucket sort. k = number of digits in the maximum value.

Data Structure: 10 buckets (one per digit 0-9). Array rebuilt after each digit pass.

📖 Algorithm Explanations

🔑 Kruskal's Algorithm

Strategy: Sort all edges by weight. Add the cheapest edge that doesn't form a cycle.

Data Structures:
• Edge List — sorted by weight O(E log E)
• Union-Find — O(α(n)) cycle detection

Time: O(E log E) · Space: O(V)

🌱 Prim's Algorithm

Strategy: Start at one node. Grow the MST by always picking the cheapest edge connecting a visited node to an unvisited one.

Data Structures:
• Adjacency List — O(V+E) space
• Min-Heap — O(log V) extraction

Time: O(E log V) · Space: O(V+E)

📋 Array — Interactive

📋 Array — Random Access Container

Used by: All 8 sorting algorithms
Key property: Elements stored in contiguous memory. Access any element by index in O(1).

Operations: access[i] O(1) · search O(n) · insert O(n) · delete O(n)

In sorting: the array is passed to each algorithm. Bubble/Selection/Insertion sort in-place. Merge Sort creates auxiliary arrays for merging.

📚 Stack — LIFO (Last In, First Out)

Used by: Recursion (call stack), Quick Sort's recursive calls
Key property: Only the top element is accessible. Push = add to top. Pop = remove from top.

Operations: push O(1) · pop O(1) · peek O(1)

Every function call in recursion pushes a new frame onto the call stack. When the function returns, the frame is popped.

🚶 Queue — FIFO (First In, First Out)

Used by: BFS graph traversal (foundation for Prim's concept)
Key property: Enqueue at back, dequeue from front. Contrast with Stack.

Operations: enqueue O(1) · dequeue O(1) · peek O(1)

A Priority Queue (Min-Heap) is a special queue where the item with the smallest key exits first — used by Prim's algorithm.

🔗 Singly Linked List

Used by: Foundation for dynamic stacks/queues and adjacency lists
Key property: Nodes linked by pointers. No random access — must traverse from head.

Operations: access O(n) · insert at head O(1) · search O(n)

Adjacency lists (used in MST algorithms) are often implemented as arrays of linked lists.

⛰️ Min-Heap — Priority Queue

Used by: Prim's Algorithm (always picks cheapest border edge)
Key property: Every parent ≤ both children. Root is always the minimum.
Stored as an array: parent of index i is at ⌊(i-1)/2⌋; children are at 2i+1 and 2i+2.

Operations: insert O(log n) · extract-min O(log n) · peek-min O(1)

🕸️ Graph — Adjacency List

Used by: Both Kruskal's and Prim's MST algorithms
Representation: Map<vertex → list of (neighbor, weight)>
Space: O(V + E). Iteration over neighbors: O(degree(v)).

vs. Adjacency Matrix: Matrix uses O(V²) space but O(1) edge lookup. Adjacency list is preferred for sparse graphs (few edges vs V²).

🔀 Union-Find (Disjoint Set Union)

Used by: Kruskal's Algorithm — detects whether adding an edge would form a cycle
Operations:
• find(x) — returns root of x's component. Path compression makes the tree nearly flat.
• union(x,y) — merges x and y's components. Union by rank keeps the tree shallow.

Both operations run in O(α(n)) ≈ O(1) amortized — effectively constant time.
